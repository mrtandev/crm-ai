คู่มือการใช้งานระบบ: AI Animal Monitoring System (System 4)

ระบบเฝ้าระวังสัตว์ป่าด้วยปัญญาประดิษฐ์ (Al Animal Monitoring System)

คู่มือการใช้งานเทคโนโลยีต้นแบบปัญญาประดิษฐ์สำหรับการประมวลผลจากกล้อง

**จัดทำโดย:** ภูวนาถ เผือกทอง

1. ข้อมูลทั่วไปของระบบ

ภาพรวมเทคโนโลยีและส่วนประกอบหน้าจอ (รูปที่ 1-1)

ระบบ Al Animal Monitoring System เป็นซอฟต์แวร์ที่ใช้รับภาพจากกล้องมาเพื่อเฝ้าระวังการบุกรุกของสัตว์ป่า โดยโปรแกรมพัฒนาบนพื้นฐานภาษา **Python** และมีปัญญาประดิษฐ์แบบ **Realtime Object Detection** ติดตั้งไว้อยู่ด้วย

รูปที่ 1-1 ส่วนประกอบหน้าจอของโปรแกรม

ส่วนประกอบหน้าจอหลัก (รายละเอียด)

1 ส่วนที่ 1: **Control Bar** (ปุ่มกดสำหรับควบคุมการทำงานของโปรแกรม)
2 ส่วนที่ 2: **Config Section** (ปรับแต่งผลการตรวจจับและเงื่อนไขการแจ้งเตือน)
3 ส่วนที่ 3: **Image Screen** (แสดงภาพจากกล้อง ผลของการตรวจจับ และใช้สำหรับการวาด Alert Region)
4 ส่วนที่ 4: **Status Bar** (ส่วนสำหรับแสดงสถานะของการทำงานของโปรแกรม)

2. การทดสอบและการใช้งานระบบ

2.1-2.2 การเริ่มต้นและการป้อนข้อมูล (รูปที่ 2-2)

ในการเริ่มต้นใช้งานให้เปิดโปรแกรมขึ้นมา และทดสอบระบบโดยการกดปุ่ม **Open Video File** เพื่อนำเข้าวีดีโอทดสอบ โดยจะมีวีดีโอขึ้นที่ Image Screen

รูปที่ 2-2 การทำงานของโปรแกรมในการแสดงผลภาพและการตรวจจับ

  • สำหรับการใช้งานจริงกับกล้องที่เป็นฮาร์ดแวร์ภายนอก ให้กดปุ่ม **Stop Monitoring** ก่อนแล้วกดปุ่ม **Start Webcam**
  • ปุ่ม Open Video File ใช้เพื่อเปิดหรือนำเข้าวีดีโอจากภายนอกสำหรับการทดสอบ

2.3 โหมด Line Crossing (การข้ามเส้น) (รูปที่ 2-4)

  • เลือก **Alert region mode** เป็น **Line Crossing** แล้วลากเมาส์ขีดเส้นแบ่งเขตที่ต้องการ
  • **เงื่อนไขแจ้งเตือน**: หากส่วนใดส่วนหนึ่งของกรอบการตรวจจับ **(Bounding Boxes)** มีการข้ามเส้นที่กำหนดไว้
  • **การแจ้งเตือน**: หน้าจอจะเปลี่ยนเป็นสีแดงและ Status Bar จะแสดงคำว่า **!!! REGION BREACH !!!**

รูปที่ 2-4 ผลการตรวจจับและการแจ้งเตือนผ่านหน้าจอ

2.4 โหมด Polygon Area (เชิงพื้นที่) (รูปที่ 2-6)

  • เลือก **Alert Region Mode** เป็น **Polygon Area** แล้วทำการคลิกจุดบน Image Screen เพื่อสร้าง Alert Area
  • **เงื่อนไขแจ้งเตือน**: จะมีการแจ้งเตือนเมื่อ **จุดศูนย์กลาง (Centroid)** ของวัตถุอยู่ในขอบเขตของ Polygon Area ที่กำหนดเอาไว้

รูปที่ 2-6 การตรวจจับด้วย Alert Polygon Area

3. การขยายโมเดลการตรวจจับ

3.1 โมเดลการตรวจจับในปัจจุบัน

ในปัจจุบัน โมเดลสำหรับการตรวจจับใช้สำหรับการตรวจจับการบุกรุกของ **กระทิงในพื้นที่โดยเฉพาะ** หากต้องการสร้างโมเดลตรวจจับอื่น ๆ สามารถทำเพิ่มได้โดยกรุณาติดต่อผู้ดูแลระบบ

3.2 การเชื่อมต่อระบบระยะไกล (API Integration)

หากต้องการให้โปรแกรมส่งการเชื่อมต่อผ่านระบบระยะไกล จะต้องทำการเชื่อมต่อระบบ Al Animal Monitoring System เข้ากับ **ระบบ API** ของแพลตฟอร์มที่ต้องการส่งการแจ้งเตือน

การปรับแต่งเพิ่มเติมและบันทึก Config

  • ผู้ใช้สามารถปรับ **Confidence Threshold** ตามต้องการได้
  • Config ของ Alert Region สามารถบันทึกและโหลดได้ด้วยปุ่ม **Save Region Config** และ **Load Default Config** ตามลำดับ
