คู่มือการใช้งานระบบ: AI Visual Label Reader (System 5)

ระบบอ่านฉลากของชิ้นส่วนของงานประกอบด้วยปัญญาประดิษฐ์

คู่มือการใช้งานเทคโนโลยีต้นแบบปัญญาประดิษฐ์สำหรับการประมวลผลจากกล้อง

**จัดทำโดย:** ภูวนาถ เผือกทอง

1. ข้อมูลทั่วไปของระบบ

ภาพรวมและวัตถุประสงค์ (Al Visual Label Reader)

ระบบ Al Visual Label Reader เป็นระบบซอฟท์แวร์ที่ใช้รับภาพจากกล้องมาเพื่ออ่านหรือแปลความของตัวอักษรบนฉลากวัตถุ โดยมุ่งเน้นสำหรับงานที่รับภาพฉลากของชิ้นส่วนในอุตสาหกรรมที่ต้องมีการนำชิ้นส่วนต่าง ๆ เข้ามาประกอบด้วยกัน

เป้าหมายของระบบ

  • ทำให้การประกอบชิ้นงานในโรงงานทำได้รวดเร็วมากขึ้น
  • แก้ปัญหาเรื่องของการจัดการคลังชิ้นส่วน

โครงสร้างและเทคโนโลยีหลัก

  • ระบบใช้ปัญญาประดิษฐ์ร่วมกับภาพจากกล้อง/รูปภาพ นำมาอ่านข้อความบนฉลากด้วยเทคนิค **Large Language Model**
  • ระบบจะทำการแยกข้อมูลพร้อมสร้าง **QR Code** สำหรับนำไปสืบค้นหรือบันทึกในฐานข้อมูล
  • ส่วนประกอบแบ่งเป็น 2 ส่วน: ส่วนของการประมวลผลภาพ และส่วนแสดงผลข้อมูลติดต่อกับผู้ใช้ (**Graphical User Interface: GUI**)
  • ส่วนติดต่อของผู้ใช้ถูกออกแบบให้เป็นแบบ **Web-based GUI** เพื่อให้เข้าถึงได้ง่ายจากหลายอุปกรณ์และประหยัดต้นทุนฮาร์ดแวร์

Workflow Overview บน n8n Server (รูปที่ 1-1)

การติดตั้งระบบประมวลผลส่วนกลางจะกระทำผ่านซอฟท์แวร์ **n8n** โดยมี Workflow Overview ดังต่อไปนี้:

รูปที่ 1-1 N8N WORKFLOW สำหรับประมวลผลภาพที่ติดตั้งบน SERVER

  1. โมเดลที่ใช้ประมวลผลจะทำงานผ่าน Ollama Platforms
  2. รอรับ **Web Hook** มาจากส่วนที่ติดต่อการทำงานร่วมกับผู้ใช้
  3. นำภาพมาประมวลผลจากนั้นจึงส่งผลลัพธ์กลับไปที่ส่วนของ User Interface

2. การทดสอบและการใช้งานระบบ

2.1 หน้าจอเริ่มต้นและตัวอย่างผลลัพธ์ (รูปที่ 1-2, 2-1)

เมื่อเปิด GUI ขึ้นมา หน้าจอเริ่มต้นโปรแกรมจะแสดงดังตัวอย่างด้านล่างนี้ และหลังจากประมวลผลเสร็จสิ้น ผลลัพธ์ที่ได้จะถูกแสดงเป็นตารางพร้อม QR Code

รูปที่ 1-2 ตัวอย่างหน้าตาโปรแกรมของส่วนติดต่อกับผู้ใช้ (GUI)

2.2-2.5: การประมวลผลผ่านการอัพโหลดไฟล์ภาพ

  • ตั้งค่าระบบบน Server ให้เป็น **Auto Activate** และเปิด GUI ขึ้นมาผ่านเว็บ Browser (หากใช้งานบนเครือข่ายเดียวกันสามารถเข้าถึงได้ผ่าน URL ของ Server)
  • หากต้องการประมวลผลไฟล์ภาพ ให้กดที่ Tab **Upload** แล้วกด **Select file** เพื่อเลือกอัพโหลดรูปภาพที่ต้องการประมวลผล (**รูปที่ 2-2**)

    รูปที่ 2-2 การเลือกอัพโหลดไฟล์รูปภาพ

  • เมื่อทำการอัพโหลดไฟล์เรียบร้อย ภาพที่อัพโหลดจะแสดงในส่วน Preview (**รูปที่ 2-3**)

    รูปที่ 2-3 ตัวอย่างภาพที่อัพโหลดไปแล้ว

  • เมื่อพร้อม ให้กดปุ่ม **Submit Image for Analysis** (**รูปที่ 2-4**) เพื่อส่งภาพไปประมวลผลที่ Server

    รูปที่ 2-4 การ SUBMIT IMAGE

  • รอผลการประมวลผลโดยดูสถานะจาก **Status Bar** (**รูปที่ 2-5**)

    รูปที่ 2-5 การแสดงสถานะประมวลผลข้อมูลของโมเดล

  • เมื่อประมวลผลเสร็จสิ้น (สถานะ "**Analysis completed successfully!**") (**รูปที่ 2-6**) ให้เลื่อนลงไปดูข้อมูลที่ถูกประมวลผลและคัดแยกมา และดู **QR Code** ที่ถูกสร้าง (**รูปที่ 2-7**)

    รูปที่ 2-6 สถานะของการประมวลผลที่เสร็จสิ้น

    รูปที่ 2-7 ตัวอย่างของข้อมูลที่ประมวลผลมาเรียบร้อยพร้อม QR CODE

2.6-2.7: การประมวลผลผ่านการถ่ายภาพจากกล้อง

  • หากต้องการถ่ายภาพจากกล้อง ให้เลือก Tab **Capture** (**รูปที่ 2-8**)

    รูปที่ 2-8 การเลือกโหมดถ่ายภาพ

  • กด **Start Camera** เพื่อทำการบันทึกภาพด้วยการกดปุ่ม **Take Snapshot** (**รูปที่ 2-9**)

    รูปที่ 2-9 การถ่ายภาพจากกล้อง

  • หากต้องการถ่ายใหม่ให้กดปุ่ม **Retake Image**
  • เมื่อได้ภาพแล้ว ให้ทำตามกระบวนการของการ Submit Image ก็จะประมวลผลรูปภาพได้เช่นเดียวกับกระบวนการของการอัพโหลดรูปภาพ

    • เอกสารคู่มือระบบ Al Visual Label Reader (System 5)
