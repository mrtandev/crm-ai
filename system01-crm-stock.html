ระบบที่ 1: Line OA Bot + CRM & Stock Automation

ผสาน Line OA Chatbot, vTiger CRM และ Loyverse POS ด้วย n8n เพื่อตอบคำถามลูกค้า, คัดกรอง Lead, และตรวจสอบสต็อกสินค้าแบบ Real-time

1. บทนำ: การเปลี่ยน Line OA ให้เป็นเครื่องมือสร้าง Lead อัตโนมัติ

ระบบนี้เป็นหัวใจสำคัญของการดำเนินงาน โดยใช้ AI Chatbot ตอบคำถามพื้นฐาน และเมื่อลูกค้าถามถึงสินค้า/สต็อก, Chatbot จะเชื่อมต่อกับระบบ Loyverse POS (หน้าร้าน) เพื่อดึงข้อมูลสต็อกจริงมาตอบ จากนั้นจะบันทึกข้อมูลลูกค้าที่สนใจลง vTiger CRM ทันที ซึ่งช่วยลดงานซ้ำซาก (AE) และไม่พลาดโอกาสในการขาย (MM)

2. วัตถุประสงค์หลัก (Object of Expenditure)

MM (Make Money): ไม่พลาดโอกาสในการขาย (Lead) และมีข้อมูลลูกค้าพร้อมปิดการขาย
ST (Save Time): ลดเวลาในการตอบคำถามและการเช็คสต็อกด้วยคน
AE (Avoid Effort): หลีกเลี่ยงการคีย์ข้อมูลลูกค้าซ้ำซากและการบันทึก Lead เข้า CRM
MC (More Comfort): ลูกค้าได้รับคำตอบสต็อกสินค้าที่แม่นยำ 24/7

3. เครื่องมือที่ใช้ (Technology Stack)

  • **Line OA:** ช่องทางรับ Lead และตอบคำถามลูกค้า
  • **n8n Community:** เครื่องมือ Automation Core สำหรับเชื่อมต่อทุกระบบ
  • **AI Model API:** (เช่น Dialogflow, Gemini) ใช้สำหรับ Chatbot Core และ Lead Qualification
  • **Loyverse POS:** ระบบ POS หน้าร้าน (ใช้ API เพื่อดึงข้อมูล Inventory/Stock)
  • **vTiger CRM:** ระบบ CRM หลัก (ใช้ API เพื่อบันทึก Lead/Contact)

⚙️ Workflow: Line OA Lead & Stock Automation

Step 1: Node 1: Line OA Webhook Trigger

**Node 1: Line**

**Trigger Type:** **Webhook**
**Events:** Message (รับข้อความทุกประเภท)

สิ่งที่ต้องทำ: กด 'Activate' และคัดลอก Webhook URL ไปวางใน Line Developer Console

Step 2: Node 2: AI วิเคราะห์ Intent (Stock Check, Interested)

**Node 2: HTTP Request (หรือ AI Node)**

**Method:** POST | **Header:** `Authorization: Bearer YOUR_AI_KEY`
**Prompt Structure:** `"วิเคราะห์ข้อความ {{$json.events[0].message.text}} แยก Intent เป็น STOCK_CHECK หรือ LEAD_GEN และดึง Product Keyword ตอบกลับเป็น JSON..."`

Data Mapping: ใช้ Expression: {{$json.events[0].message.text}} เพื่อดึงข้อความที่ลูกค้าพิมพ์มา

**เป้าหมาย:** ให้ AI แยกแยะว่าลูกค้าสนใจเช็คสต็อก หรือสอบถามข้อมูลเพื่อเป็น Lead

Step 3: Node 3: IF Node (แยกตาม Intent)

**Node 3: IF**

**Condition 1 (Output 1: Stock):** **{{$json.intent}}** `equals` **STOCK_CHECK**
**Condition 2 (Output 2: Lead):** **{{$json.intent}}** `equals` **LEAD_GEN**

**เป้าหมาย:** แยก Workflow ออกเป็น 2 สาย โดย Output 1 คือ Stock Check และ Output 2 คือ Lead Generation

Step 4a: (สาย Stock Check) ดึง Inventory จาก Loyverse

**Node 4: HTTP Request (Loyverse API) - เชื่อมจาก Output 1**

**Method:** GET | **Credential:** Loyverse Token
**URL:** (Loyverse Inventory Endpoint)
**Query Param:** `item_name` = {{$json.product_keyword}}
Step 5a: (สาย Stock Check) ตอบผล Stock กลับ Line OA

**Node 5: Line**

**Operation:** Reply Message
**Reply Token:** {{$json.events[0].replyToken}}
**Text:** `"สินค้า {{$json.product_keyword}} คงเหลือในสต็อก: {{$json.stock_count}} ชิ้น ขอบคุณครับ/ค่ะ!"`

**Data Mapping:** ใช้ Expression ดึง `replyToken` และ `product_keyword` จาก Node ก่อนหน้า และดึง `stock_count` จากผลลัพธ์ Loyverse API

Step 4b: (สาย Lead Gen) สร้าง Lead ใน vTiger CRM

**Node 4: vTiger CRM - เชื่อมจาก Output 2**

**Operation:** Create
**Resource:** Lead
**Fields:**
- `Lead Source`: **Line OA**
- `Last Name`: **{{$json.events[0].source.userId}}** (ใช้ Line User ID เป็นชื่อชั่วคราว)
- `Description`: **สนใจสินค้า: {{$json.product_keyword}}**
Step 5b: (สาย Lead Gen) ตอบยืนยันกลับ Line OA

**Node 5: Line**

**Operation:** Reply Message
**Reply Token:** {{$json.events[0].replyToken}}
**Text:** `"ขอบคุณที่สนใจครับ/ค่ะ! ทีมงานของเราได้รับข้อมูล Lead แล้ว จะรีบติดต่อกลับเพื่อดูแลสินค้า {{$json.product_keyword}} ให้นะครับ!"`

5. คู่มือการเตรียมเครื่องมือเบื้องต้น (Setup Guide) **[ทำตามได้ทันที]**

รายละเอียดการเตรียม API Key และ Credentials ที่ต้องใช้ในการเชื่อมต่อ vTiger CRM, Loyverse POS และ Line OA

1. เตรียม vTiger CRM Credentials

**เป้าหมาย:** สร้าง Credential เพื่อให้ n8n สามารถเชื่อมต่อและสร้าง Lead/Contact ใน vTiger CRM ได้

ขั้นตอนที่ 1.1: ค้นหา Access Key ใน vTiger
  • ล็อกอินเข้า vTiger CRM
  • ไปที่ **My Preferences** (มักอยู่ใต้รูปโปรไฟล์)
  • ค้นหา **Access Key** และคัดลอกค่านี้มาเก็บไว้ (สำคัญมาก)
ขั้นตอนที่ 1.2: เชื่อมต่อบัญชีใน n8n [Credentials]
  • ใน n8n ไปที่เมนู **"Credentials"** และเลือก **"vTiger CRM"**
  • ใส่ **URL** ของ vTiger Instance ของคุณ (เช่น `https://yourcompany.vtiger.com`)
  • ใส่ **Username** (มักเป็น Email) และ **Access Key** ที่คัดลอกมา
  • กด **"Save"** Credential นี้จะใช้ใน Workflow (Node vTiger CRM ใน Step 4b)

2. เตรียม Loyverse POS API Key

**เป้าหมาย:** สร้าง Token เพื่อให้ n8n สามารถดึงข้อมูลสินค้าคงคลัง (Inventory) จาก Loyverse ได้

ขั้นตอนที่ 2.1: สร้าง Personal Access Token
  • ล็อกอินเข้าสู่ **Loyverse Back Office**
  • ไปที่ **Settings > Integrations > Personal Access Tokens**
  • คลิก **"Generate Token"** และคัดลอก Token ที่สร้างขึ้นมา
ขั้นตอนที่ 2.2: การใช้งานใน n8n [Header]
  • ใน **Node HTTP Request** (Loyverse API ใน Step 4a)
  • กำหนด **Headers** เป็น `Authorization` และค่าเป็น `Bearer YOUR_LOYVERSE_TOKEN`

3. เตรียม Line OA Webhook และ Channel Token

**เป้าหมาย:** ตั้งค่า Line OA ให้ส่งข้อความลูกค้ามาที่ n8n และอนุญาตให้ n8n ตอบกลับ

ขั้นตอนที่ 3.1: Line OA Channel Access Token
  • สร้าง Line Credentials ใน n8n โดยใช้ **Channel Access Token (long-lived)** จาก Line Developer Console
ขั้นตอนที่ 3.2: Line OA Webhook URL
  • ใน n8n เปิด Workflow แล้วกด **"Active"** เพื่อให้ Node Line OA Webhook Trigger สร้าง URL
  • คัดลอก URL นี้ไปวางใน **Line Developer Console > Messaging API > Webhook URL**
  • **เปิดใช้งาน Webhook** เพื่อให้ Line OA ส่งข้อความลูกค้ามาที่ n8n
