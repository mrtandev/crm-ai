1.1.1 นิยามและความแตกต่าง: No-Code (AppSheet) vs. Low-Code (n8n/GAS) (ทักษะ: AppSheet, n8n, Google App Script)

No-Code (Citizen Developers): เครื่องมือที่เน้นความเร็วในการสร้าง **Application Interface** โดยใช้การลากวาง (Drag-and-Drop) และตั้งค่าผ่าน UI เช่น **AppSheet**

Low-Code (Pro-Developers / IT): เครื่องมือที่ให้ความสามารถในการสร้าง **Workflow Logic** ที่ซับซับซ้อนขึ้น เช่น **n8n** และ **Google App Script (GAS)**

1.1.2 The Role of AI in Low-Code / No-Code (แนวคิด AI Agent และ Gemini)

AI (เช่น **Gemini API**) เข้ามาเพิ่ม "สมอง" ใน Ecosystem นี้ โดยเฉพาะใน **GAS/Low-Code** เพื่อจัดการ Unstructured Data (เช่น การสรุป Feedback ลูกค้า, การจัดหมวดหมู่ข้อความ)

2.1.1 แนวคิดหลัก (Normalization 1NF, 2NF, 3NF): (ทักษะ: Google Sheets)

Normalization คือกระบวนการจัดโครงสร้างข้อมูลในฐานข้อมูล (GGS) ให้มีประสิทธิภาพสูงสุด เพื่อลดความซ้ำซ้อน (Redundancy) และเพิ่มความถูกต้อง (Integrity)

ระดับ ความหมาย (สำหรับ Google Sheets) เหตุผลที่ต้องทำ (ป้องกันปัญหา)
1NF **Atomic Value (ข้อมูลห้ามแบ่ง)**: ห้ามมีข้อมูลหลายค่าในเซลล์เดียว (Multi-valued Attributes) และห้ามมี Merged Cells ป้องกันไม่ให้ AppSheet หรือ Automation Tool อ่านค่าได้ไม่ครบถ้วน (ต้องรู้ว่า 1 เซลล์ = 1 ข้อมูลเสมอ)
2NF **Full Dependency (ขึ้นอยู่กับ PK ทั้งหมด)**: ข้อมูลที่ไม่จำเป็นต้องขึ้นอยู่กับ Primary Key (PK) ทั้งหมด **ต้องถูกแยกออก** ไปอยู่ตาราง Master ป้องกันปัญหาความซ้ำซ้อนและการอัปเดตข้อมูลที่ผิดพลาด (เช่น ถ้าเปลี่ยนราคาสินค้า ต้องแก้หลายแถว)
3NF **No Transitive Dependency (ไม่มีข้อมูลซ้อนข้อมูล)**: ข้อมูลที่ไม่ขึ้นอยู่กับ PK โดยตรง แต่ขึ้นอยู่กับคอลัมน์อื่นที่ไม่ใช่คีย์ **ต้องถูกแยกออก** ไปอยู่ตาราง Master ทำให้การจัดการข้อมูล Master Data ง่ายขึ้น และไม่ต้องเก็บข้อมูลคำอธิบายที่ไม่จำเป็นซ้ำ ๆ

*การทำ Normalization เหล่านี้ทำให้ฐานข้อมูล **Google Sheets** ของเราพร้อมสำหรับการเชื่อมต่อกับ AppSheet และ N8N อย่างมีประสิทธิภาพ*

2.1.2 Primary/Foreign Key (PK/FK) ใน AppSheet Ecosystem: (ทักษะ: AppSheet)

Primary Key (PK): หัวใจของตาราง **ต้องไม่ซ้ำและไม่เป็นค่าว่าง** (ใช้ Unique ID Generator เช่น GAS Script)

Foreign Key (FK) และ Ref Type: คอลัมน์ FK จะถูกกำหนดเป็นชนิดข้อมูล **'Ref'** ใน AppSheet ซึ่งช่วยให้ AppSheet สามารถสร้าง **Virtual Columns** เพื่อดึงข้อมูลจากตาราง Master มาใช้งานได้

2.1.3 การจัดการ Data Validation และ Virtual Tables: (ทักษะ: Google Sheets)

Data Validation: ใช้ใน GGS เพื่อควบคุมการป้อนข้อมูล เช่น **List from a range** เพื่อดึง Dropdown list จากตาราง Master

Virtual Tables/Reports: การใช้ฟังก์ชัน QUERY() ใน GGS เพื่อรวมข้อมูลจากหลายตาราง/Sheet มาแสดงเป็น Report หรือ Dashboard

จุดเน้น: การสร้างโครงสร้างข้อมูลที่ 'ยืดหยุ่น' และ 'สอดคล้อง' กับการทำงานของ Automation.

// ชื่อไฟล์: AutoIDGenerator.gs
// คำอธิบาย: ฟังก์ชันหลักที่ทำงานเมื่อมีการแก้ไขข้อมูลใน Google Sheet (Trigger 'On Edit')

function onEdit(e) {
  // กำหนดชื่อชีทและคอลัมน์เป้าหมาย
  const TARGET_SHEET_NAME = 'Customers';
  const ID_COLUMN_INDEX = 1; // คอลัมน์ A
  const FIRST_DATA_ROW = 2; // แถวเริ่มต้นของข้อมูล (ข้าม Header)

  const range = e.range; 
  const sheet = range.getSheet(); 
  const sheetName = sheet.getName(); 

  // ตรวจสอบเงื่อนไขการทำงาน
  if (sheetName !== TARGET_SHEET_NAME || range.getRow() < FIRST_DATA_ROW) {
    return; // ไม่ใช่ชีทเป้าหมาย หรือเป็นแถว Header
  }
  
  try {
    // ใช้ Lock Service เพื่อป้องกันการชนกันของ ID เมื่อมีผู้ใช้หลายคน
    const lock = LockService.getScriptLock();
    // พยายาม Lock ภายใน 15 วินาที
    if (lock.tryLock(15000)) { 
      
      const idCell = sheet.getRange(range.getRow(), ID_COLUMN_INDEX);
      // ตรวจสอบว่า ID ได้ถูกสร้างไปแล้วหรือไม่
      if (idCell.getValue() === "") {
        const newId = generateUniqueId('CUST-');
        idCell.setValue(newId);
        Logger.log(`สร้าง ID ใหม่สำเร็จ: ${newId}`);
      }
      
      // ปลด Lock เมื่อทำงานเสร็จ
      lock.releaseLock();
    } else {
      Logger.log("ไม่สามารถ Lock ได้ภายใน 15 วินาที");
    }

  } catch (error) {
    Logger.log(`ERROR ในการสร้าง Auto ID: ${error.toString()}`);
    // สามารถเพิ่ม MailApp.sendEmail เพื่อแจ้งเตือน IT Support ได้
  }
}

// ฟังก์ชันช่วยในการสร้าง Unique ID
function generateUniqueId(prefix) {
  // รูปแบบ ID: PREFIX-YYMMDD-HHMMSS-RANDOM_HEX
  const now = new Date();
  const year = now.getFullYear().toString().substring(2);
  const month = String(now.getMonth() + 1).padStart(2, '0');
  const day = String(now.getDate()).padStart(2, '0');
  const hours = String(now.getHours()).padStart(2, '0');
  const minutes = String(now.getMinutes()).padStart(2, '0');
  const seconds = String(now.getSeconds()).padStart(2, '0');

  // ส่วน Random String (4 หลัก Hex)
  const randomHex = Math.random().toString(16).substring(2, 6).toUpperCase(); 

  const timestamp = `${year}${month}${day}-${hours}${minutes}${seconds}`;
  return `${prefix}${timestamp}-${randomHex}`;
}
